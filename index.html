1. JavaScript Async

JavaScript is single-threaded, meaning it can only execute one operation at a time. Asynchronous programming allows it to handle operations that take time without blocking other operations. 

                            
// Example: Asynchronous operation using setTimeout
console.log('Start');

setTimeout(() => {
    console.log('Async operation');
}, 2000);

console.log('End');

2. Callback

A callback is a function passed to another function as an argument, which is then invoked after the main function finishes execution. 

                            
// Example: Callback function
function doTask(callback) {
    console.log('Task started');
    callback(); // Invoke the callback
}

doTask(() => {
    console.log('Task completed');
});

3. Promise

A Promise is an object representing the eventual completion or failure of an asynchronous operation. It has three states: pending, fulfilled, or rejected. 

                            
// Example: Promise
let myPromise = new Promise((resolve, reject) => {
    let success = true;
    if (success) {
        resolve('Operation successful');
    } else {
        reject('Operation failed');
    }
});

myPromise
    .then(result => console.log(result))
    .catch(error => console.error(error));

4. Async/Await

`async` and `await` provide a cleaner way to work with Promises. An `async` function always returns a Promise, and `await` pauses the function execution until the Promise is resolved. 

                            
// Example: Async/Await
async function fetchData() {
    try {
        let result = await myPromise;
        console.log(result);
    } catch (error) {
        console.error(error);
    }
}

fetchData();

5. Event Loop

The Event Loop allows JavaScript to perform non-blocking operations, even though it's single-threaded, by offloading operations like I/O to the browser or Node.js APIs. 

                            
// Example: Event Loop
console.log('First');
setTimeout(() => console.log('Third'), 0);
console.log('Second');
